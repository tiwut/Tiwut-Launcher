# PyToExePro

Py-to-EXE-Pro – Benutzerhandbuch

Version 1.1

Willkommen bei Py-to-EXE-Pro! Dieses Handbuch führt Sie durch alle Funktionen der Anwendung, damit Sie mühelos Ihre Python-Skripte in eigenständige Windows-Programme (.exe) umwandeln können.

1. Einführung: Was ist Py-to-EXE-Pro?

Py-to-EXE-Pro ist eine grafische Benutzeroberfläche (GUI) für das mächtige Kommandozeilen-Tool PyInstaller. Es wurde entwickelt, um den EXE-Erstellungsprozess so einfach und transparent wie möglich zu gestalten, insbesondere für Entwickler, die sich nicht mit der Kommandozeile auseinandersetzen möchten.

Mit nur wenigen Klicks können Sie:

  • Ihr Python-Skript auswählen.
  • Wichtige Optionen wie das Ausgabeformat und ein benutzerdefiniertes Icon festlegen.
  • Den Erstellungsprozess live mitverfolgen.
  • Am Ende eine fertige .exe-Datei erhalten, die Sie mit anderen teilen können.

2. Systemvoraussetzungen

Damit Py-to-EXE-Pro ordnungsgemäß funktioniert, muss Ihr System Folgendes aufweisen:

  • Windows Betriebssystem (7, 8, 10, 11)
  • Python muss installiert und zum System-PATH hinzugefügt sein. (Dies ist eine Standardoption bei der Python-Installation).
  • pip, der Python-Paketmanager, muss funktionsfähig sein.

3. Schritt-für-Schritt-Anleitung

Die Anwendungsoberfläche ist in logische Schritte unterteilt.

Schritt 1: Systemprüfung und Abhängigkeiten

Bevor Sie beginnen können, muss das PyInstaller-Tool auf Ihrem System installiert sein.

  1. Statusanzeige: Direkt nach dem Start prüft die Anwendung, ob PyInstaller verfügbar ist.
    • Grün (Ready to build!): Alles ist bereit. PyInstaller wurde gefunden.
    • Rot (PyInstaller not found!): PyInstaller fehlt. Sie können nicht fortfahren, bis es installiert ist.
  2. Button "Show Install Instructions": Ist der Status rot, klicken Sie auf diesen Button. Es öffnet sich ein Fenster mit einfachen Anweisungen:
    • Öffnen Sie eine Eingabeaufforderung (CMD) oder PowerShell.
    • Führen Sie den Befehl pip install pyinstaller aus.
    • Starten Sie Py-to-EXE-Pro nach der Installation neu.

Schritt 2: Ihr Skript auswählen

  1. Python Script (.py): Dies ist das Herzstück Ihrer Anwendung.
  2. Browse...: Klicken Sie auf diesen Button, um den Dateiexplorer zu öffnen und Ihr Haupt-Python-Skript auszuwählen (z.B. main.py oder app.py). Der Pfad wird automatisch in das Textfeld eingetragen.

Schritt 3: Build-Optionen konfigurieren

Hier legen Sie fest, wie Ihre .exe-Datei erstellt werden soll.

  • Output Type:
    • One-File (.exe): Erstellt eine einzelne, eigenständige .exe-Datei. Dies ist ideal für die einfache Weitergabe. Der erste Start kann etwas länger dauern, da die Dateien temporär entpackt werden müssen.
    • One-Directory (Folder): Erstellt einen Ordner, der die .exe-Datei zusammen mit allen Abhängigkeiten (DLLs, etc.) enthält. Diese Methode hat oft eine schnellere Startzeit.
  • Console:
    • Windowless (for GUI): Wählen Sie dies für grafische Anwendungen (z.B. mit Tkinter, PyQt, CustomTkinter). Es wird kein schwarzes Konsolenfenster im Hintergrund angezeigt.
    • Show Console Window: Wählen Sie dies für Skripte, die Textausgaben produzieren oder keine eigene grafische Oberfläche haben. Wenn Sie diese Option für ein GUI-Programm wählen, erscheint ein Konsolenfenster im Hintergrund, was für das Debugging nützlich sein kann.
  • Icon File (.ico):
    • Browse...: Wählen Sie eine Icon-Datei im .ico-Format aus, um Ihrer Anwendung ein professionelles Aussehen zu verleihen.
    • Clear: Entfernt das Icon aus der Auswahl.

Schritt 4: Das Executable erstellen

  1. BUILD .EXE: Sobald alle Einstellungen vorgenommen wurden, klicken Sie auf diesen Button, um den Prozess zu starten. Der Button wird während des Prozesses deaktiviert.
  2. Fortschritt & Log:
    • Progress Bar: Zeigt Ihnen grob an, in welchem Stadium sich der Build-Prozess gerade befindet.
    • Log Window: Hier sehen Sie die Live-Ausgabe von PyInstaller. Dies ist extrem nützlich, um den Fortschritt detailliert zu verfolgen und um Fehlermeldungen zu sehen, falls etwas schiefgeht.
  3. Open Output Folder: Sobald der Prozess erfolgreich abgeschlossen wurde, wird dieser Button aktiv. Ein Klick darauf öffnet den dist-Ordner, der Ihre fertige `.exe`-Datei enthält. Dieser Ordner wird im selben Verzeichnis wie Ihr ursprüngliches Python-Skript erstellt.

4. Fehlerbehebung

Manchmal läuft nicht alles nach Plan. Hier sind Lösungen für häufige Probleme:

  • Problem: Meine erstellte .exe-Datei wird von meinem Antivirenprogramm als Virus markiert.
    Lösung: Dies ist ein häufiger "False Positive"-Fehler. PyInstaller bündelt ein Skript mit einem Interpreter, was einige Antivirenprogramme misstrauisch macht. Wenn Sie dem Code vertrauen, legen Sie eine Ausnahme in Ihrem Antivirenprogramm an. Für eine zweite Meinung können Sie Ihre .exe-Datei auf VirusTotal.com (https://www.virustotal.com/) hochladen.
  • Problem: Die .exe-Datei startet und schließt sich sofort wieder.
    Lösung: Das bedeutet fast immer, dass Ihr Skript einen Fehler hatte. Erstellen Sie die .exe-Datei neu, aber wählen Sie diesmal die Option "Show Console Window". Wenn Sie die .exe-Datei nun ausführen, bleibt das Konsolenfenster lange genug geöffnet, um die Fehlermeldung zu lesen.
  • Problem: Die .exe meldet einen ModuleNotFoundError, obwohl das Modul auf meinem PC installiert ist.
    Lösung: PyInstaller erkennt manchmal "versteckte" Importe nicht. Sie müssen PyInstaller manuell anweisen, dieses Modul mit aufzunehmen. Dies ist leider nicht über die GUI möglich. Sie müssten PyInstaller von der Kommandozeile aus mit der Option --hidden-import MODULNAME ausführen.
  • Problem: Meine .exe kann ihre zugehörigen Dateien (Bilder, Konfigurationsdateien) nicht finden.
    Lösung: Wenn eine .exe-Datei ausgeführt wird, ist ihr "Arbeitsverzeichnis" nicht unbedingt dasselbe wie das Ihres Skripts. Relative Pfade wie img/logo.png funktionieren nicht mehr. Sie müssen absolute Pfade verwenden. Ein bewährtes Code-Snippet, um den Pfad zur .exe zu finden, ist: 
    import sys
import os

if getattr(sys, 'frozen', False):
    # Wenn als .exe ausgeführt
    application_path = os.path.dirname(sys.executable)
else:
    # Wenn als .py-Skript ausgeführt
    application_path = os.path.dirname(__file__)

# Beispiel für den Zugriff auf eine Datei
path_to_logo = os.path.join(application_path, 'data', 'logo.png')
    Sie müssen die zusätzlichen Dateien (z.B. den data-Ordner) manuell in den dist-Ordner neben Ihre .exe kopieren.
